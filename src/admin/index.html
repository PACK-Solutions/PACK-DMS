Help & examples

Provide a JSON array of rule objects. This replaces the entire ACL for the document.

Field Description
principal_type Required. "user" or "role"
principal_id UUID of the user. Required when principal_type is "user".
role Role name (e.g. "admin", "editor"). Required when principal_type is "role".
permission Required. One of "read", "write", or "admin".

Examples

// Grant read to a specific user
[
  {
    "principal_type": "user",
    "principal_id": "a1b2c3d4-0000-0000-0000-000000000001",
    "permission": "read"
  }
]

// Grant admin to a role + write to a user
[
  {
    "principal_type": "role",
    "role": "admin",
    "permission": "admin"
  },
  {
    "principal_type": "user",
    "principal_id": "a1b2c3d4-0000-0000-0000-000000000002",
    "permission": "write"
  }
]
